# Internationalization (i18n) Guidelines

**Last Updated**: 2025-12-10
**Purpose**: Guide for managing multi-language documentation
**Keywords**: i18n, internationalization, 国際化, localization, ローカライゼーション, translation, 翻訳, multi-language, 多言語, Japanese, 日本語, English, 英語

## Language Support

**Primary Languages**:
- Japanese (`ja/`)
- English (`en/`)

**Draft Language**: Any (use `draft/` for work-in-progress)

---

## Directory Structure

```
~/Data1/Docs/
├── ja/                    # Japanese documentation
│   ├── README.md
│   └── [topic]/
│       └── [document].md
│
├── en/                    # English documentation
│   ├── README.md
│   └── [topic]/
│       └── [document].md
│
└── draft/                 # Language-agnostic drafts
    └── [draft].md
```

**Rule**: Same filename across language directories

## File Naming Strategy

### Parallel Structure

```
✅ GOOD:
ja/guides/installation.md
en/guides/installation.md

❌ BAD:
ja/guides/インストール.md
en/guides/installation.md
```

**Why**: Easier to track corresponding files

### Path Consistency

```
ja/[category]/[filename].md
en/[category]/[filename].md
```

**Example**:
```
ja/reference/api-reference.md
en/reference/api-reference.md
```

## Translation Workflow

### Pattern 1: Japanese First

```
1. Write Japanese version
   ja/guides/new-feature.md

2. Mark for translation
   # Add to translation queue (if using one)

3. Translate to English
   en/guides/new-feature.md

4. Review both versions

5. Commit together
   git add ja/guides/new-feature.md en/guides/new-feature.md
   git commit -m "docs: add new feature guide (ja, en)"
```

### Pattern 2: English First

```
1. Write English version
   en/guides/new-feature.md

2. Translate to Japanese
   ja/guides/new-feature.md

3. Review both versions

4. Commit together
```

### Pattern 3: Draft First

```
1. Write in draft/ (either language)
   draft/new-feature.md

2. Review and finalize

3. Translate to other language

4. Move both to production directories
   draft/new-feature.md → ja/guides/new-feature.md
                       → en/guides/new-feature.md
```

## Translation Guidelines

### Japanese to English

**Technical Terms**:
```
✅ Keep technical terms in English when appropriate
❌ Translate common English tech terms to Japanese katakana

Examples:
API → API (not エーピーアイ)
Git → Git (not ギット)
Commit → commit (not コミット in English docs)
```

**Tone**:
```
Japanese: です・ます調 (polite)
English: Professional, direct
```

**Code Comments**:
```rust
// Japanese version
// ユーザー情報を取得する関数
fn get_user() -> User { ... }

// English version
// Function to retrieve user information
fn get_user() -> User { ... }
```

### English to Japanese

**Technical Terms**:
```
✅ Use katakana for well-known English terms
✅ Keep code/command examples in English

Examples:
File → ファイル
Directory → ディレクトリ
Function → 関数
API → API (not アプリケーションプログラミングインターフェース)
```

**Politeness Level**:
```
✅ です・ます調 for user-facing docs
✅ である調 for technical specs (optional)
```

## Metadata Strategy

### Document Frontmatter

**Japanese Version** (`ja/guides/installation.md`):
```markdown
# インストールガイド

**最終更新**: 2025-12-10
**ステータス**: 公開済み
**翻訳**: [English version](../../en/guides/installation.md)

---
```

**English Version** (`en/guides/installation.md`):
```markdown
# Installation Guide

**Last Updated**: 2025-12-10
**Status**: Published
**Translation**: [日本語版](../../ja/guides/installation.md)

---
```

### Cross-Language Links

**Pattern**:
```markdown
For Japanese version, see [日本語版](../../ja/[category]/[filename].md)
For English version, see [English version](../../en/[category]/[filename].md)
```

## Synchronization

### Keeping Translations in Sync

**Version Tracking**:

Add version metadata when content diverges:

```markdown
# Japanese version
**ドキュメントバージョン**: v1.2
**対応英語版**: v1.2

# English version
**Document Version**: v1.2
**Corresponding Japanese**: v1.2
```

**When Content Diverges**:

```markdown
# Japanese version (updated to v1.3)
**ドキュメントバージョン**: v1.3
**対応英語版**: v1.2 (要更新)

# English version (still v1.2)
**Document Version**: v1.2
**Corresponding Japanese**: v1.3 (needs update)
```

### Update Checklist

When updating documentation:

- [ ] Identify which language version to update first
- [ ] Make changes to first language
- [ ] Mark corresponding version as "needs update"
- [ ] Translate/update other language version
- [ ] Verify both versions are synchronized
- [ ] Update "Last Updated" dates
- [ ] Commit both versions together

## Language-Specific Content

### When Translations Don't Make Sense

**Example 1: Japanese-specific tutorial**
```
ja/tutorials/japanese-input-method.md  ← Japanese only
en/tutorials/ ← No direct translation
```

**Example 2: English-only external links**
```
# Japanese version
詳細は[公式ドキュメント](https://example.com/docs)（英語）を参照してください。

# English version
For details, see [official documentation](https://example.com/docs).
```

### Cultural Adaptation

**Not Just Translation**:
```
Japanese: Use Japanese-style examples (名前、住所、etc.)
English: Use English-style examples (names, addresses, etc.)
```

**Example**:

**Japanese** (`ja/guides/user-registration.md`):
```markdown
## ユーザー登録の例

- 名前: 山田太郎
- メール: taro.yamada@example.jp
```

**English** (`en/guides/user-registration.md`):
```markdown
## User Registration Example

- Name: John Doe
- Email: john.doe@example.com
```

## Code Examples Strategy

### Keep Code Consistent

```rust
// Both languages should have identical code
fn main() {
    println!("Hello, world!");
}
```

### Comments in Code

**Japanese Version**:
```rust
// メイン関数
fn main() {
    // "Hello, world!" を出力
    println!("Hello, world!");
}
```

**English Version**:
```rust
// Main function
fn main() {
    // Print "Hello, world!"
    println!("Hello, world!");
}
```

**String Literals**: Keep in English unless demonstrating i18n

## Git Commit Messages

### Language Indicators

**Pattern**: `docs([lang]): description`

**Examples**:
```
✅ docs(ja): add installation guide
✅ docs(en): update API reference
✅ docs(ja,en): add new feature guide
✅ docs: fix typos across all languages

❌ add docs
❌ update
```

### When Committing Translations

```
Commit separately:
git commit -m "docs(ja): add installation guide"
git commit -m "docs(en): add installation guide (translation)"

Or together:
git add ja/guides/installation.md en/guides/installation.md
git commit -m "docs(ja,en): add installation guide"
```

## Quality Assurance

### Translation Checklist

- [ ] Terminology is consistent
- [ ] Technical terms handled appropriately
- [ ] Code examples are identical (except comments)
- [ ] Links point to correct language versions
- [ ] Tone is appropriate for target language
- [ ] Cultural examples are adapted
- [ ] Metadata is updated (Last Updated, Version)
- [ ] Cross-language links are bidirectional

## Future Expansion

### Adding New Languages

**Steps**:

1. Create new language directory: `es/`, `fr/`, etc.
2. Update root README.md with new language
3. Translate essential documents first
4. Gradually expand coverage
5. Update this guide with new language patterns
